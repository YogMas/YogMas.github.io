# 二分

二分即每次将剩下的数组划分成左边一半和右边一半，判断答案在左边一半还是右边一半，然后选择一半的数组，并不断重复上述操作

大部分操作需要保证数组有序

int find(int x,int l,int r) //返回x的下标
{
    while(l <r)
    {
        int mid = l + r + 1>> 1;
        if(checked) r = mid - 1;
        else l = mid;
    }
}

int find(int x,int l,int r) //返回x的下标
{
    while(l <r)
    {
        int mid = l + r >> 1;
        if(checked) l = mid + 1;
        else r = mid;
    }
}

需要注意的是是否在算 mid 的时候需要加上 1

如果 left 是往 mid+1 跳转的话 则需要 如何 right 是往 mid-1 跳转的话则不需要

因为涉及到边界问题，我们知道 C++ 中的除法都是整除会向下取整，如果 right 是 mid -1 的话 不断往 mid 的左边跳转 如何跳转到了最左边 就会超出边界，想象一下 mid 是 0 的时候 还能 mid-1 吗，所以这里需要向上取整，保证不会取到 0

同理，left 是 mid +1 的话如何向下取整

# 位运算

  • 求 x 的第 k 位数组: x >>k & 1

  • 返回 x 的最后一位 1: lowbit(x) = x & -x

  • 求二进制数中最后一个 1 出现的位置（前两个操作复合而成）

int lowbit(int x)
{
    for(int i = 0 ; i < 32 ; i++)  //32为int(4 bytes 32 bits)的默认存储位数
    {
        if((x >> i) & 1)
            return i;
    }
}
//计算机中采用补码存储 补码存储将会使其所有位翻转 再末尾加1 这样就只有最后1位1 编程补码后不会变  其他的1 都会变成0

# 区间合并

将多个区间其中重合的区间进行合并

void merge(vector<int> &segs)
{
    vector<int> res;
    sort(segs.begin(),segs.end());  //按左端点先排序
    int left = -2e9,right = -2e9;
    for(auto seg:segs)
    {
        if(seg.first > right) //此区间的左端点 大于上一个合并区间的右端点 则不能合并 将上一个区间push到新数组中,并以该区间重新计算
        {
            res.push_back({left,right});
            left = seg.first;
            right = seg.second;
        }else {
            right = max(seg.second,right); //更新右端点将区间合并
        }
    }
    if(left != -2e9)
        	res.push_back({left,right});  //将最后一组push进去
    segs = res;
}

# Trie 字典树

Trie 字典树又叫前缀树 (prefix tree)，用以较快速地进行单词或前缀查询

//使用链状数组模拟链表
int son[N][26],cnt[N],index;

void insert(string s)
{
    int p = 0;
    for(int i = 0; i < s.size();i++)
    {
        if(!son[p][s[i] - 'a']) son[p][s[i] - 'a'] = ++index;
        p = son[p][s[i] - 'a'];
    }
    cnt[p]++;
}

int query(string s)
{
    int p = 0;
    for(int i = 0;i < s.size();i++)
    {
        if(!son[p][s[i] - 'a']) return 0;
        p = son[p][s[i] - 'a'];
    }
    return cnt[p];
}

# 并查集

并查集通常用来维护多个集合，从一个集合中找出一个树的根节点，并将集合中的树由根节点不断向下延伸，这颗树上的元素都是一个集合中的元素

一般是通过查找两个值所在数对应的根节点，判断两个根节点是否相同，相同则在同一个集合中

// parent[x]:表示x的父节点的下标,根节点的下标为他自己本身
int find(int x)   //找到x元素的根节点  
{
    if(parent[x] != x) parent[x] = find(parent[x]); //优化操作,在找根节点的同时,将所有已查找元素的父节点换成根节点,提高下次查找的效率
    else
    return parent[x];
}

# 字符串哈希

常用来判断两个字符串哈希

我们使用 ASCII 码来表示字符，我们能否用一个数字来表示字符串呢，显然可以

首先如果不考虑排序的问题，我们可以将字符串的字符 ASCII 码加起来，

考虑排序问题，我们可以将第 k 位乘以 N 的 k 次方

但是这样仍然可能重复，当 N 取 131 的时候，几乎不会重复

#include <iostream>

using namespace std;

const int N =100010;

const int P = 13331;

char str[N];

int h[N],p[N];

int find(int l,int r)
{
    return h[r] - h[l-1] *p[r-l +1 ];
}

int main()
{
    int n,m;
    cin >> n >> m >> str +1;
    p[0] =1;
    for(int i = 1; i <= n;i++) //对字符串进行哈希
    {
        p[i] = p[i-1] * P;
        h[i] = h[i-1] * P + str[i];
    }
    while(m--)
    {
        int l1,r1,l2,r2;
        scanf("%d %d %d %d",&l1,&r1,&l2,&r2);
        if(find(l1,r1) == find(l2,r2))
        puts("Yes");
        else
        puts("No");
    }
            return 0;
}

# 背包问题

背包问题分成 4 种背包 0-1 背包 多重背包 完全背包，分组背包

# 0-1 背包

有 N 件物品和一个容量是 V 的背包。每件物品只能使用一次。

第 i 件物品的体积是 vi，价值是 wi。

求解将哪些物品装入背包，可使这些物品的总体积不超过背包容量，且总价值最大。
输出最大价值。

第一步进行状态表示，使用 f[i][j] 表示从前 i 个物品中选，总体积不超过 j 的所有选法中的价值最大值，

如何进行集合划分呢，从前 i 个物品中选，可以分成选第 i 个物品和不选第 i 个物品，由于取价值的最大值，所以 f[i][j] = max(f[i-1][j],f[i][j]) , 显然我们无法不知道 f[i][j] 值，因此我们可以曲线救国，如果说 f[i][j] 比 f[i-1][j] 更大，那么他就可以装下第 i 个物品，且总体积是小于 j 中的最大值，那么我们去除掉第 i 个物品后，显然总体积也是小于 j-w[i] 中的最大值，那么状态转移方程可以写成 f[i][j] = max(f[i-1][j],f[i][j-v[i]]+w[i])

# 多重背包

有 N 种物品和一个容量是 V 的背包，每种物品都有无限件可用。

第 i 种物品的体积是 vi，价值是 wi。

求解将哪些物品装入背包，可使这些物品的总体积不超过背包容量，且总价值最大。
输出最大价值。
